🐞🐞 Bug 反馈指南

欢迎您为本项目提交 Bug 反馈！请严格按照下述流程和模板操作，以便我们快速定位和修复问题。

一、反馈流程

  1. 登录 GitHub 账号，访问本仓库的 Issues 页面。
  2. 点击右上角 "New Issue" 按钮。
  3. 选择 "🐞🐞 Bug Report" 模板，完整填写所有问题相关信息。
  4. 强烈建议：上传 截图、报错日志、复现代码 等附件，帮助开发者定位问题。
  5. 提交 Issue 后，可在下方留言补充信息或与开发者互动跟进。
温馨提示：
  • 请勿提交重复问题，先搜索是否已有类似 Issue
  • 请勿在同一 Issue 混报多个无关问题
  • 请保持沟通礼貌，感谢您的理解与配合！

  • 如何上传图片/日志？
    在 GitHub Issue 编辑器中，直接拖拽图片或文本文件到输入框即可自动上传。
  • 如何知道我的 Issue 是否被处理？
    关注 Issue 回复或标签变化，开发者会在处理进度时留言或变更 Issue 状态。
